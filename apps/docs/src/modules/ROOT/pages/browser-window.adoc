= BrowserWindow Component

The BrowserWindow component allows you to create authentic-looking browser mockups in your documentation, perfect for showcasing web applications, APIs, or websites.

== Overview

The BrowserWindow component mimics a real browser window with:

* macOS-style window controls (red, yellow, green dots)
* Address bar displaying the URL
* Hamburger menu icon
* Content area for your content or iframe

== Usage Patterns

=== 1. Inline Macro

For simple URL displays with iframes:

----
browser-window:https://example.com[]
browser-window:https://example.com[height=400]
----

=== 2. Block Macro

For URLs with custom height:

----
browser::https://docs.example.com[height=500]
----

=== 3. Delimited Block

For custom content instead of iframes:

----
[browser-window, url="https://app.example.com", height=350]
--
Your custom content goes here.
Can include any AsciiDoc markup.
--
----

== Best Practices

=== When to Use

* ‚úÖ Showcasing web applications
* ‚úÖ Demonstrating API responses  
* ‚úÖ Displaying code examples in context
* ‚úÖ Creating visual mockups
* ‚úÖ Educational tutorials

=== When NOT to Use

* ‚ùå For simple code blocks (use regular source blocks)
* ‚ùå For non-web content
* ‚ùå When the browser context isn't relevant

=== Tips

. **Choose appropriate heights**: Consider your content when setting height
. **Use meaningful URLs**: The address bar URL should relate to your content
. **Keep it responsive**: The component works on mobile, but test your content
. **Combine with admonitions**: Great for tips, warnings, or notes about the shown content
. **Image sizing**: Images automatically scale to fit the browser window width
. **Alt text**: Always provide descriptive alt text for accessibility
. **File formats**: Use web-optimized formats (PNG, JPG, WebP) for best performance
. **Image context**: Add captions or descriptions to explain what the image shows

== Styling and Theming

The BrowserWindow component automatically adapts to light and dark themes. It uses CSS custom properties for easy customization:

[source,css]
----
.doc .browser-window {
  /* Customize border color */
  border-color: var(--your-border-color);
}

.doc .browser-window-header {
  /* Customize header background */
  background: var(--your-header-bg);
}
----

== Troubleshooting

=== Component Not Rendering

1. Ensure the extension is properly registered in `antora-playbook.yml`
2. Check that the extension file exists at the specified path  
3. Rebuild your UI bundle if you modified CSS
4. Verify AsciiDoc syntax is correct

=== Styling Issues

1. Clear your browser cache
2. Rebuild the Antora UI bundle
3. Check CSS custom properties are defined
4. Verify no conflicting styles

=== Content Not Displaying

1. For iframe content: Check the URL is accessible
2. For custom content: Verify AsciiDoc syntax
3. Check browser console for errors
4. Ensure height is sufficient for your content
