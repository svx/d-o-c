= AsciiDoc Linter

Welcome to the AsciiDoc Linter documentation!

The AsciiDoc Linter is a minimal, extensible tool for linting AsciiDoc documents. It helps ensure consistency and quality in your documentation by checking for common issues and style violations.

== Features

* **Extensible rule system**: Easy to add custom rules
* **Configurable**: Customize rules and their settings
* **Fast**: Built with TypeScript and Node.js for performance
* **CLI and programmatic API**: Use from command line or integrate into your build process

== Quick Start

.Installation
[source,bash]
----
pnpm add @testthedocs/asciidoc-linter
----

.Basic usage
[source,bash]
----
adoc-lint my-document.adoc
----

== Browser Window Component

The documentation includes a BrowserWindow component that mimics browser interfaces, perfect for showcasing web applications, APIs, or websites.

=== Basic Usage

You can use the browser window component with different syntaxes:

.Inline macro (for URLs only)
[source,asciidoc]
----
browser-window:https://example.com[]
----

.Block macro with content
[source,asciidoc]
----
browser::https://docs.example.com[height=300]
----

=== Examples

Here are some live examples of the BrowserWindow component:

.Simple URL display (inline macro)
browser-window:https://d-o-c-docs.localhost:8081[]

.API Documentation with custom height
browser::https://api.example.com/docs[height=400]

.Browser window with custom content block
[browser-window, url="https://app.example.com", height=350, title="Custom App"]
--

----

.Another content example

* User statistics

=== Advanced Usage

You can customize the browser window with various attributes:

* `url` - The URL shown in the address bar
* `height` - Minimum height of the component (default: 300px)  
* `title` - Accessible title for screen readers

.Complex example with code
